How to load our model

We have modified our code following the guidelines you give. The function srmodel() has been put in arch.py and the checkpoint.pth has been included according to the guidelines. To run our code, you should extract models/,arch.py and checkpoint.pth from our submitted zip file, and put them in the same directory with your test code: 

|--- models/
|--- arch.py
|--- checkpoint.pth
|--- your-test-code.py
in your-test-code.py you can write: 
from arch import srmodel
model = srmodel()
to load our model.

Why our model should be loaded directly by model = torch.load('checkpoint.pth')

Essentially, our Track 1 model is in models/rfdn_half/RFDNB4S.py and Track 2 model is in models/rfdn_half/RFDNB2S.py. You can load them by 

from models.rfdn_half.RFDNB4S import RFDNB4S_P
model = RFDNB4S_P() # Track 1

from models.rfdn_half.RFDNB2S inport RFDNB2S_P
model = RFDNB2S_P() # Track 2
But since we applied auto-pruning to our model, state dict in checkpoint.pth could not be loaded into the original model. That is why you should directly load the model by model = torch.load('checkpoint.pth')(as is written in our arch.py).
